# 文档风格美化

优化 Markdown 文档的排版、结构和视觉呈现。

## 排版规范

### 标题层级
- `#` 仅用于文档标题（全文唯一）
- `##` 用于一级章节
- `###` 用于二级子章节
- 不超过3级深度

### 列表
- 无序列表用 `-`，二级缩进2空格
- 有序列表用 `1. 2. 3.`
- 列表项不超过2行，过长改用段落

### 表格
- 必须包含表头行
- 对齐：文字左对齐、数字右对齐
- 复杂表格添加 `|:---|:---:|--:|`

### 代码
- 代码块标注语言：`python`, `bash`, `json`, `yaml` 等
- 行内代码用 `` ` `` 包裹
- 文件路径用 `` ` `` 包裹

### 强调
- 粗体 `**重点**`：用于关键结论
- 斜体 `*术语*`：用于新引入的术语
- 删除线 `~~删除~~`：用于废弃内容
- 不混用多种强调

### 链接和引用
- 外链：`[文字](完整URL)`
- 内部引用：使用锚点 `[见上文](#章节名)`
- 参考链接：文末集中管理

## 美化检查清单
- [ ] 标题不超过3级深度
- [ ] 表格有表头且对齐正确
- [ ] 代码块标注了语言
- [ ] 无孤立的单行段落
- [ ] 段落长度适中(3-6行)
- [ ] 全文档风格统一
