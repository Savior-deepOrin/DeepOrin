# 文档自审

以审稿人/编辑视角审视文档，找出逻辑、结构和表达上的问题。

## 审查流程

### 第一遍：结构审查
1. 扫标题层级 — 结构是否合理
2. 看各节长度 — 是否比例失衡
3. 检查逻辑流 — 前后是否连贯
4. 标出缺失内容 — 缺什么

### 第二遍：段落审查
1. 每段是否有中心句
2. 支撑句是否充分
3. 段间过渡是否自然
4. 有无跑题内容

### 第三遍：句子审查
1. 过长句子（>40词）需要拆分
2. 模糊措辞替换为具体表达
3. 重复表达合并或删除
4. 检查语法和拼写

### 第四遍：数据审查
1. 所有数字是否准确
2. 引用是否完整
3. 声明是否有证据支撑
4. 前后数据是否一致

## 输出格式

```
## 审查意见

### 结构问题（严重）
- [ ] 问题描述 → 建议修改

### 逻辑问题（中等）
- [ ] 问题描述 → 建议修改

### 表达问题（轻微）
- [ ] 问题描述 → 建议修改

### 建议总结
3-5条最重要的修改建议
```
